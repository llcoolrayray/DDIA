#### 分区与二级索引
二级索引是关系型数据库的必备特性，不是用于唯一标识一条数据而是用来加速特定值的查询。有两种主要的方法支持对二级索引进行分区：`基于文档分区的二级索引`
和`基于词条的二级索引分区`

#### 基于文档分区的二级索引
在`基于文档分区的二级索引`中，每个分区完全独立，各自维护自己的二级索引。当查询时，会请求所有的分区，然后合并结果返回给客户端。

#### 基于词条的二级索引分区
对所有的数据构建全局索引，而不是每个分区维护自己本地的索引，并且全局索引也会分区。这种方式称为词条分区。

词条分区的优点是查询更加高效，查询时先访问索引所在的分区，然后再向数据所在的分区发起请求即可。但是缺点是当新增一条数据时十分复杂且缓慢，需要更新多个
二级索引而二级索引可能分布在多个分区之中。
